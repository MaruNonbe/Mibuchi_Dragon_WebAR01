# 長井駅到着 GPS WebAR

長井駅のGPS位置から半径100m以内で、動物弦楽四重奏モデルを表示するWebARページです。
GPSは表示可否の判定に使い、モデルはWebAR空間内に固定表示します。
表示安定性を優先し、モデル位置と向きは固定しています。
なお、iOS Safari単体ではカメラ映像から前後左右の物理移動を3D追跡するSLAMは使えないため、実際に歩いてモデルの周囲へ回り込む移動量そのものは反映しません。
カメラ背景は `getUserMedia` で取得した背面カメラ映像を全面表示し、その上に透明なA-Frameシーンを重ねています。

## 表示位置

- 対象: 山形鉄道フラワー長井線 長井駅
- 緯度: `38.106518`
- 経度: `140.033583`
- 表示半径: `100m`

座標は Wikipedia の Nagai Station (Yamagata) ページに掲載されている座標を使用しています。

## 検証用GPS

長井駅以外に、検証用としてタスパークホテル長井のGPSも追加しています。

- 対象: タスパークホテル長井 検証地点
- 緯度: `38.1013040`
- 経度: `140.0433785`
- 表示半径: `100m`

座標は OpenStreetMap / Overpass API の `タス・パーク・ホテル` POI を使用しています。

## 自宅テスト地点

開始画面の「現在地を自宅テスト地点に保存」ボタンで、端末の現在地を半径100mの表示地点として保存できます。
保存先はその端末の `localStorage` のみで、GitHubやコードには自宅座標を書き込みません。

## ファイル構成

- `index.html`: GPS WebARページ
- `main.js`: 100m判定、アナウンス、表示制御
- `styles.css`: スマホ向けUI
- `assets/nagai_station_quartet.glb`: Blenderから書き出されるモデル

## アニメーション

GLB内のアニメーション再生には、`animation-player.js` の `play-all-model-animations` コンポーネントを使用しています。
GLBに含まれる全アニメーションクリップを Three.js の `AnimationMixer` で直接ループ再生します。
SafariでGLBクリップが再生されない場合に備え、弓・頭・音符・文字・音波はノード名を使ってWeb側でも直接アニメーションさせます。

## 音楽

音楽は外部の既存曲や音源ファイルを使わず、`main.js` の Web Audio API で生成するオリジナルループです。曲調は爽やかな駅到着イベント向けで、表示中の駅の編成に合わせて弦楽、木管、金管、打楽器寄りの音色へ切り替わります。iOS / Android の自動再生制限に対応するため、音楽は画面上の「音楽」ボタンをタップしたあとに再生されます。

## GLBを書き出す

Blenderで次を実行します。

```bash
blender --python ../create_animal_string_quartet.py
```

実行すると、次のファイルが自動生成されます。

```text
gps-webar/assets/nagai_station_quartet.glb
```

BlenderのText Editorからスクリプトを実行する場合も、同じ場所に書き出されます。

## 公開条件

iOS / Android の実機では、次が必要です。

- HTTPSで公開すること
- カメラ許可
- 位置情報許可
- iOSではモーション/方位の許可

ローカルPCの `file://` ではカメラやGPSが動かないことがあります。GitHub Pages、Netlify、Firebase Hostingなど、HTTPSで配信できる場所に `gps-webar` フォルダを配置してください。

## テスト

長井駅から100m以内:

- 「長井駅到着」のアナウンス
- 動物弦楽四重奏モデル表示

100mより外:

- 距離表示のみ
- モデル非表示

GPS精度が悪い場合、100m以内でも一時的に表示されないことがあります。駅構内や屋外で数秒待つと安定しやすいです。
